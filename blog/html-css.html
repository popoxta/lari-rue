CSS Positioning

Within the abyssal reaches of CSS reside enigmatic beasts that hold dominion over the positioning of elements - the formidable relative, the ethereal absolute and the unwavering fixed. But what allows these creatures the governance of position? Take note, traveler, for their mastery is merely veiled simplicity, and all returns to the normal flow of the labyrinthine script.

Normal Flow

The term "normal flow," also known as document flow or default flow, refers to the default behavior of how HTML elements are placed onto a webpage when no styling properties are applied. (This is also where position:static is applied!)

When elements are in a normal flow state, they are typically displayed one after the other in a top-down fashion for block-level elements or side by side for inline-level elements.

When elements are set to position:absolute or position:fixed, the element in question is whisked away from the normal document flow. This allows for these elements to be positioned in ways or places that would otherwise be very tricky to achieve.

The most commonly used positions are relative, fixed and absolute. I will briefly explain each, including their use cases and how they relate to one another.

Relative

When an elementâ€™s position is set to position:relative, it is still positioned as per the normal flow of the document - however it may then be offset from this position. Properties such as left, right, top and bottom may be used to relatively position the element.

It is important to note that with the relative position, the space which the element would have occupied in a static state is still considered occupied by this element - even if it has been positioned to an entirely different sector of the document.

Relatively positioned elements are commonly used for creating offsets, decorative effects or as a parent container to absolutely positioned elements.

Absolute

An absolute-positioned element is removed entirely from the normal document flow, and in contrast to position:relative, no space is kept in the original static position.

An element with position:absolute becomes positioned relatively to its closest positioned parent element, commonly with a position of relative. For example, an absolute positioned element within a relatively positioned container can be positioned relatively to its parent container.

Absolutely positioned elements are often used to create modals, overlays or tooltips.

Fixed

The functionality of a position:fixed element creates a major change from the other potential position values discussed. It ensures that the element in question is fixed to the viewport, so that even if the user scrolls down the page, the element will still be within it's designated position.

Fixed position elements are used to create persisting sidebars, action buttons, modals or alerts.

Sticky

position:sticky is a position that holds a hybrid role between fixed and relative positioning. An element with this position is rendered as per the normal flow, however when the user scrolls down the page and the threshold value for the element is met, the element becomes sticky and sticks to a specified position on the viewport.

Sticky positioned elements are used to create sticky headers or navigation menus.

Conclusion

Consider this passage a mere glimpse into their enigmatic ways. Take heed, for it is only through bold experimentation that you shall truly unravel these arts for yourself, and attain mastery of elemental positioning.

Summon forth a modal, evoke a sticky header, or wield your powers to conjure a fixed menu. Dare to cast aside the known conventions and propose a yet unspoken and unprecedented specification. May your vision guide you, traveler.
